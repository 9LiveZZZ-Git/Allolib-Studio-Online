# HRFilter integration notes

`Gamma/HRFilter.h` is a fully header-only parametric HRTF (Iida 2007/2018, see header docstring) that synthesizes the head-shadow + pinna notches/peaks from listener pose + source position — it does NOT load HRIR data from disk, does NOT depend on a SOFA file, and does NOT pull in `Gamma::SoundFile` or `Gamma::Recorder` (both excluded from `libal_web.a`). Its only includes are `Gamma/Filter.h`, `Gamma/Spatial.h`, and `Gamma/Types.h`; the companion `HRScene<Nsrc>` template additionally uses `gam::ReverbMS` (from `Gamma/Effects.h`, which transitively pulls Delay/Envelope/Filter/Noise/Oscillator). All of these are header-only and therefore already on the WASM include path via `backend/docker/compile.sh:84` (`-I"$GAMMA_DIR"`); nothing in `Gamma/external/Gamma/src/*.cpp` is needed for HRFilter, so **no edit to `allolib-wasm/CMakeLists.txt`'s `GAMMA_SOURCES` list is required**. Examples can use `#include "Gamma/HRFilter.h"` directly with the existing build, and `gam::HRFilter` / `gam::HRScene<N>` resolve against the already-linked Gamma static library. Verified by reading `allolib_playground/allolib/external/Gamma/Gamma/HRFilter.h` (216 lines, all defs inline) and confirming none of its dependency chain references the excluded Gamma .cpp files (`AudioIO.cpp`, `Recorder.cpp`, `SoundFile.cpp`).
